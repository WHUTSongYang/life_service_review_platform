# 无关话题拒答

## 说明

当**意图路由**将用户消息判为 **off_topic**（与本平台无关：如无关编程、天气、其他公司、产品政策不覆盖的个人危机等）时启用本技能。智能体**不得**调用商户 MCP 工具，也不得用 FAQ RAG 回答无关百科式问题。产品侧可用**固定短拒答**完成回复，无需再走一轮生成。

## 使用方式

- **何时使用**：路由 JSON 中 `"intent": "off_topic"`，或策略将该轮标为越界。
- **前置条件**：无（不需要 MCP）。
- **调用时机**：路由完成后立即处理；该意图下不要把店铺列表或 FAQ 段落拼进回复。

## 步骤

1. **确认意图**  
   从最新路由结果确认 `intent === "off_topic"`。

2. **不要调用 MCP**  
   不要调用 `search_shops`、`get_shop_detail`、`search_nearby_shops` 等商户工具。

3. **不要用 RAG 当事实来源**  
   除非产品策略明确要求合并平台规则，否则不要将本轮当作 FAQ 检索任务。

4. **输出拒答**  
   仅输出产品约定的拒答文案（例如一句话表示无法回答）。不要编造平台功能。

5. **安全**  
   若用户表达自伤或紧急情况，按组织危机预案处理（热线、转人工）；本技能不能替代该流程。

## 输入约定

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| intent | string | 是 | 必须为 `off_topic`。 |

路由示例：

```json
{
  "intent": "off_topic",
  "sub_intent": null,
  "keyword": null,
  "type": null,
  "shop_id": null
}
```

## 输出约定

| 字段 | 类型 | 说明 |
|------|------|------|
| reply | string | 仅固定拒答文本；无结构化店铺数据。 |

## 示例

**用户：**「用 Java 写一段快速排序。」

**路由输出：**

```json
{ "intent": "off_topic" }
```

**智能体动作：**

1. 不调用 MCP。  
2. 使用配置的拒答句（例如「我无法回答。」）。
